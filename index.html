01 · Executive Summary

What is NexReader?

NexReader is a next-generation, AI-augmented e-book and document reading platform designed natively for tablets. It bridges the gap between a personal digital library, an intelligent reading assistant, and a social knowledge platform — all wrapped in a futuristic interface that feels alive and personal.

Unlike existing readers that are glorified PDF viewers, NexReader is built from the ground up around three pillars: universal format support, AI-powered comprehension tools, and seamless cloud persistence. Every annotation, highlight, reading session, and personal insight is saved, synced, and searchable across all user devices.

Mission Statement: Empower every reader to engage more deeply with knowledge — no matter the format, no matter the device, no matter where they are in the world.

04 · Universal Format Support

Every Format, Zero Exceptions

NexReader's format parser is built on a modular rendering engine. Each format has a dedicated rendering pipeline, with a unified annotation layer sitting on top. Users should never need to know or care what format a file is — it just opens.

📖

  • EPUB Fixed Layout

  • CBZ / CBR (Comics)

  • XPS / OXPS

OCR Engine: Scanned PDFs and image-based documents are automatically processed through an on-device OCR pipeline (Vision framework on iOS, ML Kit on Android) to make them fully searchable and annotatable — even if originally just a photo of a page.

05 · AI Reading Engine

Your Intelligent Reading Companion

NexReader ships with a deeply integrated AI layer — not a chatbot bolted on, but a reading companion that understands the book you are reading and helps you engage with it more deeply. The AI engine is powered by a combination of on-device models for privacy and cloud LLMs for depth.

🧠

AI Summarisation

Instant chapter summaries, section summaries, and full-book summaries. "Catch me up" feature explains what happened before a long reading gap. Adjustable summary length and depth.

💬

Ask the Book

Natural language Q&A grounded entirely in the text. Ask "Who is the villain in chapter 5?" or "What is the author's argument about inflation?" — answers are cited with page references.

🔤

Vocabulary Intelligence

Long-press any word for instant dictionary, etymology, translation, and usage examples. "Word web" shows how a word is used across the entire book. Saves words to a personal vocabulary list.

🗺️

Character & Entity Maps

Automatically builds and updates a character relationship map as you read. Tracks place names, organisations, dates, and events. Spoiler-safe — only shows what you've already read.

📝

AI Note Enhancement

Your rough margin notes are expanded into polished, structured summaries. AI suggests connections between your notes across different books in the library.

🎯

Reading Speed & Comprehension Coach

Tracks WPM, asks periodic comprehension check-in questions, adjusts reading recommendations. Provides personalised tips to improve reading efficiency over time.

🌐

Translation Layer

Real-time paragraph translation into 50+ languages. Dual-language view showing original and translation side-by-side. Preserves formatting and layout.

💡

Smart Recommendations

AI analyses reading habits, genres, themes in annotations, and pacing to recommend next books. Cross-references with Open Library and community lists.

Privacy First: All on-device AI features (vocabulary, OCR, basic summarisation) use Apple Neural Engine / Android NPU and never leave the device. Cloud AI features (deep Q&A, knowledge graph) are opt-in, with data processed through encrypted pipelines and not used for training. Enterprise plan includes fully on-premise AI option.

08 · Cloud & Sync Architecture

Everywhere, Always In Sync

Cloud persistence is a core promise of NexReader. Every reading position, highlight, and note must sync across all user devices within seconds — even with intermittent connectivity.

Offline-First Architecture

The app works fully offline. Books are cached locally in an encrypted on-device SQLite database (via SQLCipher). Changes queue in a local sync log and push to the cloud the moment connectivity returns. Conflict resolution uses a server-wins CRDT model for positions, and merge-all for annotations (no annotation is ever lost).

Real-Time Sync

WebSocket connection maintains a live sync channel per active session. Position updates push within 1 second of page turn. Any device that picks up the book instantly jumps to the last-read position. "Continue reading" notification on other devices when a session ends.

Cloud Storage

Book files stored in per-user encrypted S3 buckets (AWS S3 or self-hosted MinIO). CDN-served for fast downloads globally. Delta sync for annotation updates — only diffs are transmitted, never the full file. Resumable chunked upload for large PDFs and comic archives.

Backup & Recovery

Daily automated backup of user library metadata and annotations to a separate geographic region. User-triggered full backup export (ZIP with all books + annotations JSON). Point-in-time recovery for Pro and Scholar plans. 30-day retention of deleted items before permanent removal.

Infrastructure: API Gateway → Kubernetes (EKS) → Node.js / Go microservices → PostgreSQL (RDS Multi-AZ) + Redis (ElastiCache) + pgvector extension. Book files on S3 (us-east-1 primary, ap-south-1 replica for India users). CloudFront CDN for cover images and static assets. Target: 99.95% uptime SLA.

13 · Open Questions & Risks

Known Unknowns

⚠ DRM & Licensing Risk

Files purchased from Amazon/Apple are DRM-protected. We do not support DRM stripping. Users must bring their own DRM-free files or public domain content. Legal review required before claiming "any format" in marketing.

⚠ AI Cost Scaling

At scale, LLM API costs for "Ask the Book" and summarisation could be significant. Need to implement: response caching per book, rate limits on free tier, and evaluation of smaller local models for most queries.

⚠ Storage Costs

Free users with large book collections could strain S3 costs. Need to enforce storage quotas strictly, and evaluate compression strategies for storing multiple copies of the same public-domain book.

❓ Format Rendering Fidelity

Complex EPUB fixed-layout and highly formatted PDFs are notoriously difficult to render perfectly. Decision needed: license an existing rendering engine (Readium, PDF.js) vs. build in-house. Recommend licensing first.
